SQL Server 2005

 SQL Server  2005 نصب   

 

    

  پس از اجراي Autorun و ظاهر شدن برنامه نصب گزينه Sql Server 2005 Workproup را انتخاب کنيد.

 

                              

                                                        

 

 در اين صفحه شرايط و مقررات را بخوانيد و تيک قسمت ... I accept را بزنيد.وسپس بر روي next کليک کنيد.

 

 

                                                                 

                                                                     

                 

   در اين مرحله، موارد پيش نياز اين نرم افزار جهت نصب اعلام ميگردد. با انتخاب دکمه Install موجب نصب آنها ميشويم .

 

                                                                     

 

  در انتهاي نصب نيز دکمه Next را کليک ميکنيم .

 

                                                                    

 

   در اين فرم خوش آمدگويي و شروع نصب نرم افزار SQL Server 2005 را اطلاع ميدهد. دکمه Next را کليک ميکنيم.

  

                                                               

 

 صفحه پيکربندي سيستم ظاهر مي شود. در اين مرحله ،سيستم  setup کليه ي نرم افزارهاي مورد نياز و نيز تنظيمات سيستم عامل و  سرويسهاي مورد نياز را چک مي کند.اگر پيکر بندي با موفقيت به اتمام برسد،گزينه ي success  در بالاي فرم ظاهر مي شود. در اين صورت روي next  کليک کنيد.

                   

 

                 

 

Sql server 2005    داراي کامپوننت هاي مختلفي مي باشد.اين کامپوننت ها شامل سرويس هاي متنوعي جهت انجام عمليات خاص

   مي باشند که تنها در صورتي که از آنها استفاده مي شود بايد نصب گردند.بايد به اين نکته توجه داشت که هر کدام از اين سرويس ها در

  صورت نصب،مقدار قابل توجهي از قدرت سرور را به خود اختصاص مي دهند،بنابراين تنها زماني که برنامه هاي کاربردي ما از اين

  سرويس ها استفاده مي کنند بايد نصب شوند.مواردي که در عکس بالا انتخاب شده اند،کامپوننت هايي هستند که معمولا توسط برنامه

  نويسان و راهبران پايگاه داده مورد نياز هستند.

 

                  

 

   گزينه Default Instance را انتخاب نماييد.

   بايد به اين نکته توجه کنيد که بر روي يک کامپيوتر تنها يک Default Instance مي تواند وجود داشته باشد.بنابراين اگر گزينه Default Instance

  فعال است و در اين کامپيوتر SQL ديگري نيز نصب است به اين معني است که در حال حاضر يک SQL SERVER  به صورت  Default    

 نصب شده است و  در آنصورت شما بايد با يک نام ديگر (Named Instance )  نصب کنيد(که البته بهتر است SQL  قبلي را Unistall   کنيد.)

  بر روي Next  کليک کنيد.

              

 

  کليه ي سرويسهايي که بر روي سيستم عامل اجرا مي شوتد نياز به يک Account   با دسترسي هاي لازم دارند.در واقع سيستم عامل قبل

  از اجراي هرگونه سرويس،به وسيله اين  Account  از مجاز بودن آن سرويس اطمينان حاصل مي کند.از آنجايي که SQL SERVER   نيز به

 عنوان يک سرويس بر روي سيستم عامل فعاليت مي کند نياز به ارائه يک Account   مي باشيد.لازم به ذکر است در محيط هاي عملياتي

  توصيه مي شود از يک User  که در Domain تعريف شده است استفاده شود.ولي در محيط هاي برنامه بر روي Next  کليک کنيد.

 

 

   گزينه Windows Authentication را انتخاب کنيد.سپس بر روي Next  کليک کنيد.

 

 

   تنظيمات پيش فرض collation  را تغيير داده،بدين صورت که

   ابتدا SQL Collation (Used for compatibility…. را تيک بزنيد و روي 

 Dictionary order ,case-insensitive,for use with the 1256(Arabic)Character set  

   کليک نماييد.

 

 

   نرم افزار جديد شرکت Microsoft   اين قابليت را دارند که در صورت بروز مشکل خطاي ايجاد شده را براي آن شرکت ارسال کنند.در صورتي که

   مايل هستيد مي توانيد اين گزينه ها را انتخاب کنيد(من انتخاب اين گزينه ها را پيشنهاد نمي کنم) و روي Next کليک کنيد.

 

 

    صفحه پيش نصب نمايش داده مي شود.

    روي Install کليک کنيد.

 

 

   پيشرفت نصب نمايش داده مي شود.

    

 

   پس از اينکه عمليات نصب اتمام يافت روي Next کليک کنيد.

 

                                                          

                                                               

                     

 صفحه اتمام نصب SQL Server 2005 نمايش داده مي شود.روي Finish کليک کنيد.

 

 

 

    پس از نصب برنامه sql server management studio   را از منويstart اجرا مي کنيم.

 

 

             

   

   پس از نمايش فرم مربوطه نام سرور يا local  را انتخاب کرده کليک راست کرده و آيتم properties  را انتخاب مي کنيم.

 

                                                         

   پس از نمايش فرم مربوطه از تب security  گزينه sql server  and windows  را انتخاب کرده و ok  را انتخاب مي کنيم.
